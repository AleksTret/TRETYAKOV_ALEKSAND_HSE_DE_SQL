## Ответы

### 1. Учитывается ли в вашей методике возможность проводить частичное досрочное погашение?

Нет, не учитывается.
В текущей версии расширения параметр prepayments зарезервирован, но логика частичного досрочного погашения (ЧДП) не реализована. Мною было принятно решение не реализовывать эту логику для избежания увеличения объема кода. При желании функционал можно будет добавить.

### 2. Какие шаги нужно выполнить для использования вашего расширения в прикладных программах?

**Установка расширения на сервере:**

1. Установить PL/Python3 в PostgreSQL:
   ```bash
   sudo apt install postgresql-plpython3-16
   sudo systemctl restart postgresql
   ```

2. Скопировать папку `loan_annuity` на сервер.

3. Внутри папки выполнить:
   ```bash
   sudo make install
   ```

4. В psql выполнить:
   ```sql
   CREATE EXTENSION loan_annuity;
   ```

5. Использование в прикладной программе:
    - Подключиться к БД.
    - Выполнять SQL-запросы к функциям расширения:
    ```sql
    SELECT * FROM build_schedule(1, 1000000, 12, 12, '2024-01-01');
    SELECT calculate_psc(1, 1);
    SELECT export_schedule_json(1, 1);
    ```

### 3. В расширении присутствует одна таблица payment_schedules. 

#### 3.1 При использовании расширения в прикладных программах нужно будет, видимо, копировать данные их БД этой программы в эту таблицу?

Нет, копировать данные из прикладной БД в payment_schedules не нужно. Функция build_schedule сама заполняет эту таблицу при вызове. Прикладная программа передаёт параметры кредита (сумма, ставка, срок), а расширение уже сохраняет график. 

#### 3.2 А что надо было бы сделать, чтобы обрабатывать данные непосредственно в БД прикладной программы?

Нужно избавиться от жёсткой привязки к таблице payment_schedules.
Варианты:

- Переписать функцию build_schedule, чтобы она не вставляла данные, а только возвращала график, тогда таблица не нужна.

- Использовать динамический SQL (EXECUTE format(...)), чтобы имена таблиц и столбцов передавались как параметры. Это позволит работать с любой структурой прикладной БД.
```sql
sql
CREATE FUNCTION build_schedule_dynamic(
    source_table text,
    amount_col text,
    ...
) RETURNS SETOF payment_row AS $$
BEGIN
    EXECUTE format('SELECT %I FROM %I ...', amount_col, source_table);
    ...
END;
$$ LANGUAGE plpgsql;
```
